# ข้อผิดพลาดที่เจอบ่อย — สรุปจากแบบฝึกหัด C# ตอนที่ 1

เก็บไว้อ่านทบทวน เจอซ้ำเมื่อไหร่ให้เช็กลิสต์นี้ก่อน

## 1. Boundary / เครื่องหมายเทียบ (เจอบ่อยสุด)
- `>` แทน `>=` → ค่าขอบพอดีหลุด (เช่น 60 ไม่ผ่าน)
- เงื่อนไขกลับด้าน `<` แทน `>`
- เทียบผิดตัว เช่น `x > words.Length` (ควรเทียบความยาวของคำ ไม่ใช่ของลิสต์)

## 2. ตัวสะสม / ตัวแปรในลูป
- อย่าใส่ `return` ในลูปถ้าต้องสะสมค่า → return เตะออกตั้งแต่รอบแรก
- ตัวสะสม (sum / count / best) ต้องประกาศ **นอกลูป ก่อนลูป**
- อย่าเอาตัวแปรที่กำลังวน (n) มาบวกเป็นตัวนับเอง → ใช้ตัวแปรแยก
- ตัวแปรต้องให้ค่าเริ่มต้นก่อนใช้ เช่น `string best = "";`

## 3. อย่า hardcode ค่าตายตัว
- `i < 6` → ควรผูกกับตัวแปร เช่น `i <= n`
- `parts[0][0]` ทุกรอบ → ควรใช้ตัวที่กำลังวน `p[0]`

## 4. property vs method
- เมธอดต้องมี `()` เช่น `x.ToUpper()` ไม่ใช่ `x.ToUpper`
- เมธอด string คืนค่าใหม่เสมอ ต้องรับค่ากลับ ไม่ได้แก้ตัวเดิม
- อย่าใส่ตัวแปรเกินความจำเป็น (ข้อที่แค่นับ ไม่ต้องมี best)

## 5. พิมพ์ผิด / ลืม save
- C# เทียบ string ตรงตัวเป๊ะ ตัวอักษรต่างตัวเดียวก็ไม่เท่ากัน
- `Lenght` → `Length`
- **กด Ctrl+S ก่อนรันทุกครั้ง** (แก้แล้วไม่ save = รันของเก่า)
- ลบ `using` ที่ IDE เติมมาเองแต่ไม่ได้ใช้

---

## กฎทอง 4 ข้อ
1. ระวัง `>` กับ `>=`
2. ตัวสะสมอยู่นอกลูป
3. เมธอดต้องมี `()`
4. save ก่อนรัน
